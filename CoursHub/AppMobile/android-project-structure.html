// 01 — Introduction

Pourquoi autant de fichiers ?

😱

La première fois qu'on ouvre un projet Android Studio, c'est le choc. Des dizaines de dossiers, des centaines de fichiers... C'est normal de se sentir perdu.

Bonne nouvelle : tu n'as besoin de toucher qu'à 3 ou 4 fichiers pour créer ta première application. Le reste, c'est soit généré automatiquement, soit des configurations que tu ne modifieras presque jamais.

Pense à Android Studio comme une cuisine professionnelle. Tu as des ustensiles, des équipements, des réserves... mais quand tu cuisines, tu utilises juste le plan de travail et quelques outils.

// 03 — Les fichiers clés

Clique pour découvrir chaque fichier

Voici les fichiers que tu vas utiliser au quotidien. Clique sur chaque carte pour en savoir plus.

AndroidManifest.xml
La carte d'identité de ton application
Essentiel

C'est le fichier le plus important de ton projet. Il dit au système Android tout ce qu'il doit savoir sur ton app :

• Le nom du package (identifiant unique de ton app, ex: com.monnom.mamap)
• La liste de toutes tes activités (écrans)
• Les permissions dont tu as besoin (internet, caméra, GPS...)
• Quelle activité s'ouvre en premier au lancement

Chaque fois que tu crées un nouvel écran, tu dois l'enregistrer ici !

<activity android:name=".MainActivity"> <intent-filter> <!-- Ceci dit que MainActivity est l'écran de démarrage --> <action android:name="android.intent.action.MAIN"/> </intent-filter> </activity>
MainActivity.kt (ou .java)
Le cerveau de ton écran principal
À modifier

C'est ici que tu écris la logique de ton application. Chaque écran (Activity) a son propre fichier Kotlin/Java. MainActivity = l'écran principal.

C'est ici que tu mets : ce qui se passe quand on clique sur un bouton, comment récupérer du texte d'un champ, comment naviguer vers un autre écran, etc.

class MainActivity : AppCompatActivity() { override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) setContentView(R.layout.activity_main) // Ton code commence ici ! } }
res/layout/activity_main.xml
Le design visuel de ton écran
À modifier

C'est le fichier de mise en page (layout). Il définit ce que l'utilisateur voit à l'écran : les boutons, les textes, les images, les champs de saisie...

Tu peux l'éditer en mode visuel (drag & drop) ou en mode XML. Chaque vue a un android:id qui permet de la retrouver depuis le code Kotlin.

<Button android:id="@+id/monBouton" android:layout_width="wrap_content" android:layout_height="wrap_content" android:text="Clique-moi !" />
💡 ASTUCE : Le @+id/monBouton te permet ensuite de faire val btn = findViewById(R.id.monBouton) dans ton fichier .kt pour le contrôler avec du code !
res/values/strings.xml
Tous les textes de ton application
À modifier

Bonne pratique : au lieu d'écrire tes textes directement dans le XML de layout, tu les mets ici. Pourquoi ? Pour faciliter la traduction en plusieurs langues plus tard.

<resources> <string name="app_name">Mon Application</string> <string name="bouton_ok">C'est parti !</string> </resources>

Et dans ton layout, tu écris : android:text="@string/bouton_ok"

res/values/colors.xml
La palette de couleurs de ton app
À modifier

Définis ici toutes les couleurs de ton application. Comme ça, si tu veux changer ta couleur principale, tu le fais une seule fois et ça s'applique partout.

<resources> <color name="couleur_principale">#3DDC84</color> <color name="couleur_fond">#FFFFFF</color> <color name="texte_fonce">#212121</color> </resources>
app/build.gradle.kts
La liste des dépendances (bibliothèques)
Config

C'est ici que tu ajoutes des bibliothèques externes (libraries) à ton projet. Envie d'utiliser Retrofit pour les appels réseau ? Glide pour les images ? Tu les ajoutes ici.

dependencies { // Bibliothèques Android de base implementation("androidx.appcompat:appcompat:1.7.0") // Exemple : ajouter une bibliothèque d'images implementation("com.github.bumptech.glide:glide:4.16.0") }
💡 ASTUCE : Après avoir modifié ce fichier, Android Studio te demandera de faire "Sync Now" — c'est normal, il doit télécharger les nouvelles bibliothèques.
res/drawable/
Tes images et icônes
À modifier

C'est le dossier où tu mets tes images, icônes et formes. Tu peux y mettre des fichiers PNG, JPEG, WebP, ou des fichiers XML qui décrivent des formes vectorielles.

Pour utiliser une image dans ton layout : android:src="@drawable/mon_image"

💡 ASTUCE : Préfère les fichiers SVG/Vectoriel (VectorDrawable) aux images PNG — ils s'adaptent à toutes les tailles d'écran sans pixelliser !
// 04 — À éviter

Les fichiers à ne pas toucher

app/build/ (tout le dossier)
Fichiers générés à chaque compilation
Ne pas toucher

Ce dossier est entièrement généré automatiquement par Android Studio à chaque fois que tu compiles ton projet. Il contient le code compilé, les fichiers APK intermédiaires, etc.

Si tu modifies quoi que ce soit ici, tes changements seront écrasés à la prochaine compilation. Et ce dossier fait des Go — c'est pour ça qu'on le met dans .gitignore pour ne pas le sauvegarder sur GitHub.

⚠️ ATTENTION : Même si tu vois un bug "dans" un fichier de build, la solution se trouve TOUJOURS dans ton code source, pas dans build/.
.gradle/ et .idea/
Paramètres internes d'Android Studio
Ne pas toucher

.idea/ contient les paramètres de ton IDE (Android Studio). C'est propre à ta machine. .gradle/ contient le cache de Gradle (l'outil qui compile ton projet).

Si ton projet a des comportements bizarres, tu peux parfois faire "File → Invalidate Caches" pour nettoyer ces dossiers. Mais tu ne les modifies jamais manuellement.

gradlew et gradlew.bat
Scripts d'exécution de Gradle
Ne pas toucher

Ce sont des scripts qui permettent d'exécuter Gradle (l'outil de build) sans avoir à l'installer manuellement. gradlew est pour Linux/Mac, gradlew.bat pour Windows.

Ils sont générés par Android Studio et ne doivent jamais être modifiés.

res/mipmap-*/
Les icônes de l'application (plusieurs tailles)
Ne pas toucher (directement)

Ces dossiers contiennent l'icône de ton application en plusieurs résolutions (mdpi, hdpi, xhdpi, xxhdpi, xxxhdpi) pour s'adapter à tous les écrans.

Si tu veux changer l'icône, n'écrase pas ces fichiers manuellement ! Utilise plutôt : clic droit sur res → New → Image Asset — Android Studio générera automatiquement toutes les tailles.

// 05 — Ton premier projet

Schéma : Créer une app "Bonjour Monde"

Suivons ensemble les étapes pour créer une application simple avec un bouton qui affiche un message. Voilà exactement dans quel ordre travailler.

1

Créer le projet dans Android Studio

File → New → New Project → Empty Views Activity (pas Compose pour l'instant, c'est plus simple). Donne un nom à ton app, un package name (com.tonnom.tonapp), choisis Kotlin ou Java. Android Studio génère tout le squelette.

💡 PACKAGE NAME : C'est l'identifiant unique de ton app sur le Play Store. Convention : ton domaine à l'envers + nom app. Si tu n'as pas de domaine, utilise com.tonprenom.nomapp
2

Modifier le layout → activity_main.xml

Ouvre res/layout/activity_main.xml. C'est ici que tu dessines ton interface. Ajoute un TextView (texte) et un Button :

<LinearLayout ...> <TextView android:id="@+id/monTexte" android:layout_width="wrap_content" android:layout_height="wrap_content" android:text="En attente..." android:textSize="24sp" /> <Button android:id="@+id/monBouton" android:layout_width="wrap_content" android:layout_height="wrap_content" android:text="Clique ici !" /> </LinearLayout>
3

Écrire la logique → MainActivity.kt

Ouvre MainActivity.kt. Tu vas récupérer ton bouton et ton texte par leur ID, puis définir ce qui se passe au clic :

class MainActivity : AppCompatActivity() { override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) setContentView(R.layout.activity_main) // On récupère les éléments du layout par leur ID val monTexte = findViewById<TextView>(R.id.monTexte) val monBouton = findViewById<Button>(R.id.monBouton) // On définit ce qui se passe quand on clique monBouton.setOnClickListener { monTexte.text = "Bonjour le Monde ! 🎉" } } }
4

Personnaliser → strings.xml & colors.xml

Bonne pratique : met tes textes dans strings.xml et tes couleurs dans colors.xml pour garder ton code propre.

<!-- strings.xml --> <resources> <string name="app_name">Mon App</string> <string name="btn_label">Clique ici !</string> <string name="message_hello">Bonjour le Monde ! 🎉</string> </resources> <!-- colors.xml --> <resources> <color name="colorPrimary">#3DDC84</color> <color name="colorBackground">#FFFFFF</color> </resources>
5

Lancer l'application ! 🚀

Branche ton téléphone Android (ou lance un émulateur via Device Manager), puis clique sur le bouton ▶ Run en haut. Android Studio va :

① Compiler ton code (Kotlin → bytecode)
② Générer un APK (fichier d'installation)
③ Installer et lancer l'app sur ton appareil

🎉 FÉLICITATIONS ! Tu viens de créer ta première app Android. Clique sur le bouton dans l'app — le texte devrait changer !
// 06 — Récapitulatif

Ce qu'il faut retenir

✅ Fichiers que tu modifies

  • AndroidManifest.xml
  • MainActivity.kt / .java
  • res/layout/*.xml
  • res/values/strings.xml
  • res/values/colors.xml
  • res/drawable/ (tes images)
  • app/build.gradle.kts (pour les libs)

🚫 Fichiers à ignorer

  • app/build/ (tout le dossier)
  • .gradle/ (cache)
  • .idea/ (IDE settings)
  • gradlew / gradlew.bat
  • gradle/wrapper/*
  • local.properties
  • settings.gradle.kts (au début)

📐 La logique à comprendre

  • XML = ce qu'on VOIT (design)
  • Kotlin/Java = ce qui SE PASSE (logique)
  • R.id.xxx = pont entre XML et code
  • Manifest = déclaration de tout
  • build.gradle = gestion des libs

🚀 Prochaines étapes

  • Ajouter un 2e écran (Activity)
  • Naviguer entre les écrans (Intent)
  • RecyclerView pour des listes
  • Room pour une base de données locale
  • Retrofit pour les APIs REST
