📅 Historique d'Android

L'histoire d'un OS qui a conquis le monde · Survole chaque étape

🌍 En moins de 20 ans, Android est passé d'une startup inconnue au système le plus utilisé de la planète. Voici les grandes dates à retenir pour l'exam !
2003
🏢 Création d'Android Inc.

Fondée à Palo Alto, Californie par Andy Rubin, Rich Miner, Nick Sears et Chris White. À l'origine, l'idée était de créer un OS pour… appareils photo numériques, pas pour téléphones !

2005
💰 Google rachète Android Inc.

Acquisition pour environ 50 millions de dollars. Andy Rubin et son équipe rejoignent Google. Le projet passe en mode secret et Google commence le vrai développement de l'OS mobile.

2007
🤝 Open Handset Alliance

35 entreprises s'unissent : Google, Motorola, Samsung, LG, HTC, Intel, Qualcomm… Objectif : développer un OS mobile open source et gratuit. La même année, Apple lance l'iPhone 1.

2008
🚀 Android 1.0 — Le lancement

Premier smartphone Android : le HTC Dream (T-Mobile G1). L'Android Market (futur Play Store) démarre avec seulement 50 applications. La révolution commence.

2024
🏆 Android 15 — Aujourd'hui

API level 35. Plus de 3 milliards d'appareils actifs dans le monde. 70%+ de parts de marché mondial. Téléphones, tablettes, montres, voitures, TV… Android est partout.

🤔 C'est quoi Android ?

4 définitions clés · Clique sur chacune pour comprendre

+
🐧
Un OS basé sur le noyau Linux
Système d'exploitation open source · Clique !
🏠 Imagine une maison.
Le noyau Linux c'est les fondations — tu ne les vois pas mais sans elles tout s'effondre.
Android c'est toute la maison bâtie dessus : les murs, les pièces, le mobilier.

Open Source = le plan de la maison est public. Tout le monde peut le voir, le copier et l'améliorer. C'est pour ça que Samsung, Xiaomi, OnePlus… peuvent tous faire leur propre version d'Android !
+
☕
Les apps sont développées en Java / Kotlin
Le langage de programmation · Clique !
🗣️ Java c'est la langue dans laquelle tu parles à Android.
Tu ne peux pas coder directement en 0 et 1, alors Java joue le rôle d'intermédiaire.

Java → langage historique, enseigné dans ce cours
Kotlin → langage plus moderne, recommandé par Google depuis 2017. Plus concis, moins d'erreurs.

Les deux compilent vers le même format final → ils sont compatibles dans un même projet.
+
🤖
Machine Virtuelle : Dalvik / ART
Comment ton code Java tourne sur Android · Clique !
🔄 Problème : Java standard est fait pour tourner sur PC (JVM classique), pas sur téléphone.
Solution : Android a sa propre machine virtuelle optimisée !

Dalvik (ancien) → traduit le code à chaque exécution. Plus lent.
ART (actuel) → compile une fois à l'installation. Beaucoup plus rapide au lancement.

Le bytecode Dalvik (.dex) utilise 30% d'instructions en moins que la JVM standard → idéal pour les téléphones avec peu de RAM.
+
📦
Les applications = fichiers .apk
Le format de distribution d'une app Android · Clique !
📮 APK = Android Package. C'est une archive ZIP renommée en .apk.
Elle contient absolument tout ce dont l'app a besoin pour tourner :

🔹 Le code compilé en bytecode Dalvik (.dex)
🔹 Les ressources (images, layouts XML, sons)
🔹 Le AndroidManifest.xml (carte d'identité de l'app)
🔹 Les bibliothèques natives (.so)

Analogie : C'est exactement comme un colis Amazon 📦 — tout est emballé dedans, tu reçois la boîte, tu l'ouvres, tout est là.

🧩 Composants d'une Application

Les 4 briques fondamentales de toute app Android · Clique pour détailler

🏗️ Toute application Android est construite avec ces 4 types de composants. Certains tu les vois (Activities), d'autres travaillent en coulisse (Services, Broadcast Receivers, Content Providers). Tu dois obligatoirement les déclarer dans le AndroidManifest.xml !
🎭

Activities

Un écran = une Activity. C'est ce que l'utilisateur voit et avec quoi il interagit directement.
Analogie 🎬 : Une Activity = une scène dans un film.
Ton app peut avoir plusieurs scènes mais tu n'en vois qu'une à la fois.
  • Représente un écran unique de l'interface
  • Gère toutes les interactions utilisateur
  • Étend la classe Activity
  • A un cycle de vie géré par le système
  • Exemple : écran de login, liste de contacts
⚙️

Services

Tâches longues en arrière-plan, sans aucune interface utilisateur visible.
Analogie 🎵 : Un Service c'est la musique qui joue en fond.
Tu navigues dans d'autres apps mais la musique continue — tu ne "vois" pas le service.
  • Aucune interface utilisateur
  • Continue de tourner quand l'app est en fond
  • Étend la classe Service
  • Exemples : téléchargement, lecture audio, sync
📡

Broadcast Receivers

Écoute des événements du système ou d'autres apps et y réagit.
Analogie 📻 : C'est une radio qui capte des signaux.
Le système Android diffuse des "broadcasts" et ton app peut réagir.
  • Écoute les événements système (batterie, Wi-Fi…)
  • Peut recevoir des messages d'autres apps
  • Étend BroadcastReceiver
  • Peut démarrer une Activity ou un Service
💾

Content Providers

Partage de données entre applications de manière sécurisée et contrôlée.
Analogie 🏦 : C'est une banque — elle détient des données et les partage selon des règles strictes.
  • Gère et expose des données à d'autres apps
  • Utilise SQLite en interne
  • Étend ContentProvider
  • Accès sécurisé avec permissions
