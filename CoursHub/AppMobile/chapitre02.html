⚙️ La Pile d'Outils

Clique sur chaque bloc pour comprendre à quoi il sert

🏠 Imagine que tu veux construire une maison.
Tu as besoin de plusieurs outils : une perceuse, du ciment, des briques…
Pour faire une appli Android, c'est pareil. Tu as besoin d'une "pile d'outils". Chaque outil a un rôle précis. On va voir ça ensemble !
+
☕
JDK / JRE — Le moteur
Java Development Kit · Clique pour comprendre !
🚗 C'est comme le moteur d'une voiture.
Sans moteur → la voiture ne démarre pas.
Sans JDK → ton code Java ne peut pas tourner.

JDK = la boîte à outils complète (compilateur + moteur + bibliothèques)
JRE = juste le moteur (pour faire tourner, pas pour développer)
Ce que fait le JDK
// Ton code Java (MonActivite.java)
public class MonActivite { ... }

     ↓  javac compile ton code

MonActivite.class  ← bytecode (le moteur comprend ça)
+
📦
SDK Android — La boîte à outils Android
Software Development Kit · Clique pour comprendre !
🧰 C'est comme une boîte à outils IKEA.
Elle contient tout ce dont tu as besoin pour monter ton meuble.
Le SDK contient tout ce dont tu as besoin pour faire ton appli Android !
AAPT
Compile les ressources (images, XML) en un format qu'Android comprend
AVD
Un téléphone virtuel sur ton PC pour tester sans vrai téléphone
ADB
Fait le lien entre ton PC et l'émulateur (ou vrai téléphone)
+
🐘
Gradle — Le robot cuisinier
Automatise tout le processus de construction · Clique !
👨‍🍳 Gradle c'est comme un robot cuisinier (Thermomix).
Tu lui donnes les ingrédients (ton code, tes images, tes fichiers XML)…
Il fait tout tout seul : il compile, il assemble, il génère le fichier final (.apk) !

Tu n'as pas à taper des dizaines de commandes à la main. Gradle s'en occupe automatiquement 🤖
build.gradle
android {
  compileSdk 35         ← version Android cible
  defaultConfig {
    applicationId "com.exemple.monapp"
    minSdk 21             ← Android 5.0 minimum
    versionName "1.0"
  }
}
dependencies {
  // Gradle télécharge ces biblio auto !
  implementation "androidx.appcompat:appcompat:1.6.1"
}
+
💻
Android Studio — L'atelier
L'IDE où tu écris ton code · Clique !
🎨 Android Studio c'est ton atelier de travail.
Comme Word pour écrire du texte, Android Studio c'est pour écrire du code.
Mais en plus il te corrige, t'aide, et te montre à quoi ressemble ton appli en direct !

Avant → Eclipse (vieux, compliqué)
Maintenant → Android Studio (moderne, recommandé par Google ✅)

📁 Les Fichiers Créés par Défaut

Clique sur chaque fichier pour comprendre son rôle

🏘️ Imagine ton projet Android comme un quartier.
Chaque dossier / fichier a sa maison et son rôle précis.
Quand tu crées un projet, Android Studio génère tout ça automatiquement pour toi !
+
📝
MonActivite.java
src/main/java/... → Ton code
C'est ton code Java. C'est ici que tu écris la logique de ton appli.

Analogie 🎭 : C'est le scénario d'un film. Il dit quoi faire, comment réagir quand on clique.

Chaque "écran" de ton appli = 1 fichier .java (Activity)
+
🗺️
activity_main.xml
res/layout/ → L'interface visuelle
C'est le design de ton écran. Il décrit où sont les boutons, les textes, les images.

Analogie 🏠 : C'est le plan d'architecte de ta maison. Il dit où est la porte, les fenêtres…

<Button
  android:text="Clique moi !"
  android:layout_width="wrap_content"
/>
+
🔤
strings.xml
res/values/ → Les textes de l'appli
Tous les textes de ton appli sont ici.

Pourquoi pas directement dans le code ?
Parce que si tu veux traduire l'appli en arabe ou en anglais, tu changes juste ce fichier ! 🌍

<string name="app_name">Mon Appli</string>
<string name="bouton_ok">Valider</string>
+
🖼️
ic_launcher.png
res/mipmap/ → L'icône de l'appli
C'est l'icône qui apparaît sur l'écran du téléphone.

Il y en a plusieurs tailles (hdpi, xhdpi, xxhdpi…) pour s'adapter à tous les écrans.

Analogie 📸 : Comme une photo en différentes résolutions pour différents usages.
+
📋
AndroidManifest.xml
→ La carte d'identité de l'appli
LE fichier le plus important de tout ton projet.

Il dit à Android : qui je suis, quelles permissions j'ai besoin, quels écrans j'ai.

Analogie 🪪 : C'est la carte d'identité de ton appli !
Sans lui → l'appli ne peut pas s'installer.
+
⚙️
R.java
gen/ → Auto-généré ! Ne pas modifier
Généré automatiquement par Android Studio. Ne JAMAIS modifier à la main !

Il fait le lien entre ton code Java et tes ressources (images, layouts, textes).

On explique ça en détail dans la section suivante 👇

🗝️ La Classe R.java

Le concept le plus mystérieux — expliqué simplement

🗝️ R.java c'est comme un annuaire téléphonique automatique.
Tu as des ressources (images, boutons, textes)… mais ton code Java ne sait pas où elles sont !
R.java donne un numéro unique (comme un ID) à chaque ressource.
Ton code Java utilise ces numéros pour trouver les ressources. Malin non ? 😏
🗂️ Tes Ressources
📐 res/layout/interface.xml layout
🔤 strings.xml → nom_app string
🔤 strings.xml → titre_activite string
🖼️ mipmap/ic_launcher mipmap
⚡ Ce que génère R.java
← Clique sur une ressource pour voir son ID dans R.java
💡 Comment l'utiliser dans ton code Java ?

// Charger l'interface XML dans ton activité :
setContentView(R.layout.interface);
//              ↑    ↑       ↑
//          classe  type  nom du fichier sans .xml

// Récupérer un texte :
getString(R.string.nom_app);

// Récupérer une image :
getDrawable(R.mipmap.ic_launcher);

📋 AndroidManifest.xml

Clique sur chaque ligne pour comprendre ce qu'elle fait

🪪 C'est la carte d'identité + le règlement intérieur de ton appli.
Il dit au système Android : comment je m'appelle, quels droits j'ai besoin, quels écrans j'ai, lequel se lance en premier. 
<manifest package="com.exemple.bonjour"
    android:versionName="1.0">

  <uses-permission android:name="android.permission.INTERNET" />

  <application
    android:icon="@mipmap/ic_launcher"
    android:label="@string/nom_app">

    <activity android:name=".MonActivite"
      android:label="@string/titre_mon_activite">

      <intent-filter>
        <action android:name="android.intent.action.MAIN"/>
        <category android:name="android.intent.category.LAUNCHER"/>
      </intent-filter>
    </activity>
  </application>
</manifest>
👆 Clique sur n'importe quelle ligne du code pour l'expliquer
