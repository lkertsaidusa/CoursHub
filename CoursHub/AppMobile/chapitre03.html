📖 Définitions

C'est quoi une Activité ?

🎭Activité = scène de théâtre. Chaque écran = une scène = une activité. Une app contient généralement plusieurs activités. Elle est l'équivalent de la méthode main() en Java classique.

import android.app.Activity;
import android.os.Bundle;

public class ExempleActivite extends Activity {
  // ↑ HÉRITE de la classe Activity

  public void onCreate(Bundle stateSaved) {

  }
}

🔍 Log.d — Le Système de Balisage

📟Log.d = des post-it dans ton code. Tu places des messages à différents endroits → quand l'appli tourne, tu vois tout dans la console Logcat d'Android Studio. Ton meilleur ami pour déboguer ! 🕵️
Syntaxe de base
Log.d("MonTag", "Mon message");
 ↑       ↑           ↑
méthode  étiquette  message à afficher dans Logcat

// Exemple réel avec le cycle de vie :
String TAG = "CycleDeVie";           // ton étiquette

Infos utiles. Ex: "Utilisateur connecté", "Données chargées".
W

Avertissement. Quelque chose d'inhabituel, pas forcément une erreur.

📦 Bundle — Sauvegarde de l'État

Le tableau associatif typé qui sauve tes données !

🎮Problème : Score à 1500 pts. Android ferme l'appli (mémoire insuffisante). Tu reviens… score = 0. 😱
Solution : Bundle sauvegarde l'état AVANT la destruction et le restaure après.
📦 Bundle = Tableau Associatif TYPÉ

Comme un sac avec des compartiments étiquetés. Clé = String, Valeur = type précis !

"score"
→
1500
int
"nomJoueur"
→
"Alice"
String
"estConnecte"
→
true
boolean
"vitesse"
→
3.14
float
"monObjet"
→
Joueur{...}
Parcelable
Bundle est typé : le type de chaque valeur est connu.
Sauvegarder dans Bundle
@Override
protected void onSaveInstanceState(Bundle out) {
  out.putInt("score", ScoreCourant);       ← int
  out.putString("nom", nomJoueur);         ← String
  out.putBoolean("connecte", estConnecte); ← boolean
  super.onSaveInstanceState(out);
}
Récupérer depuis Bundle
@Override
protected void onRestoreInstanceState(Bundle s) {
  super.onRestoreInstanceState(s);
  ScoreCourant = s.getInt("score");       ← même clé !
  nomJoueur    = s.getString("nom");       ← même clé !
  estConnecte  = s.getBoolean("connecte"); ← même clé !
}

🔑La clé dans get doit être exactement identique (même casse) à celle dans put. Si tu fais putInt("score", 100) → tu dois faire getInt("score") et pas "Score" ni "SCORE" !

🎁 Parcelable vs Serializable — Les objets complexes

🏗️Problème : Tu as un objet Java (class Joueur avec nom, score, niveau). Comment le mettre dans un Bundle ?
Les types simples (int, String) → direct. Pour un objet complexe → il faut lui apprendre à se "démonter" et se "remonter" !
🚀 Parcelable — La voie Android

Parcelable est une interface Android qui décrit comment stocker un objet dans un Parcel (flux de données binaires ultra-rapide).

Tu dois implémenter :
• writeToParcel() → écrire les champs
• CREATOR → reconstruire l'objet

✅ Plus rapide que Serializable
✅ Recommandé sur Android
❌ Plus de code à écrire

public class Joueur implements Parcelable {
  String nom; int score;
  public void writeToParcel(Parcel out, int f) {
    out.writeString(nom);
    out.writeInt(score);
  }
  // + CREATOR pour reconstruire...
}
// Usage :
bundle.putParcelable("joueur", monJoueur);
Joueur j = bundle.getParcelable("joueur");
📦 Serializable — La voie Java

Serializable est une interface Java standard. Elle convertit l'objet en flux d'octets automatiquement.

Tu n'as presque rien à écrire :
• Juste implements Serializable
• Java fait tout le reste !

✅ Très simple à implémenter
❌ Plus lent que Parcelable
❌ Crée des objets temporaires en mémoire

public class Joueur implements Serializable {
  String nom; int score;
  // C'est tout ! Java gère le reste 😊
}
// Usage :
bundle.putSerializable("joueur", monJoueur);
Joueur j = (Joueur) bundle.getSerializable("joueur");

🤔 Quand utiliser Bundle ? (obligatoire ou facultatif ?)

✅
OUI — Données de session UI
Score, position scroll, texte saisi, onglet actif… Données temporaires liées à l'interface.
✅
OUI — Rotation d'écran !
Android détruit et recrée l'activité à la rotation. Sans Bundle → tout disparaît. Cas le plus fréquent !
❌
NON — Données persistantes
Profil, historique, paramètres → utilise SQLite, SharedPreferences ou fichiers.
⚠️
FACULTATIF — Apps très simples
Aucun état à sauvegarder ? Bundle peut être omis. Mais si l'user saisit → l'implémenter !

📊 Comparaison : Bundle vs SQLite vs Fichiers

📦 Bundle
✅Simple et automatique
❌Temporaire (session)

✅Persistant (survit réinstall)
✅Structuré (SQL)

✅Persistant
✅Photos, médias, JSON
