// 01 · définition

C'est quoi la Sécurité Informatique ?

La sécurité informatique, c'est l'ensemble des moyens mis en œuvre pour réduire la vulnérabilité d'un système contre les menaces accidentelles ou intentionnelles.

En clair : on cherche à protéger les données, les systèmes et les réseaux contre tout ce qui pourrait les compromettre. Clique sur chaque carte pour en savoir plus 👇

🦠
Menace
Type d'action susceptible de nuire…
+
Une menace représente n'importe quelle action qui pourrait nuire à un système informatique. C'est le danger potentiel, pas encore réalisé. La menace n'est pas liée à une faille particulière — elle existe dans l'absolu.
// exemples concrets
→ Un virus circule sur le réseau local
→ Un employé mécontent a accès aux serveurs
→ Une panne électrique qui peut corrompre des données
// analogie baby 🍼
C'est un voleur qui rôde dans ton quartier. Il n'a pas encore agi, mais il représente une menace. Qu'il passe à l'acte ou non, la menace existe.
Intentionnelle Accidentelle Environnementale
🕳️
Vulnérabilité
Faille ou bug exploitable…
+
Une vulnérabilité est une faille ou un bug dans un système pouvant être exploitée pour obtenir un accès illicite ou des privilèges supérieurs. C'est le chaînon manquant entre la menace et l'attaque : sans vulnérabilité, une menace ne peut pas se concrétiser.
// exemples concrets
→ Mots de passe trop simples (123456)
→ Comptes sans mot de passe
→ Logiciel non mis à jour (CVE connues)
→ Port réseau laissé ouvert inutilement
// analogie baby 🍼
C'est la fenêtre que tu as laissée ouverte chez toi. Le voleur (menace) peut s'en servir pour entrer. La fenêtre ouverte = vulnérabilité.
CVE 0-day Misconfiguration
🛡️
Contre-mesure
Actions de prévention et protection…
+
Une contre-mesure est l'ensemble des actions mises en place pour prévenir, détecter ou corriger une menace. Elle agit directement sur les vulnérabilités pour les éliminer ou les réduire. On distingue les contre-mesures techniques (antivirus, chiffrement) et organisationnelles (formation, procédures).
// exemples
→ Chiffrement des données sensibles
→ Authentification à deux facteurs (2FA)
→ Antivirus et détection d'intrusions (IDS/IPS)
→ Formation et sensibilisation des employés
→ Mise à jour régulière des systèmes
// analogie baby 🍼
Fermer et verrouiller ta fenêtre + installer une alarme + alerter les voisins = contre-mesures contre le voleur.
⚔️
Attaque
Exploitation concrète d'une faille…
+
Une attaque c'est la concrétisation d'une menace via l'exploitation d'une vulnérabilité. Tout ordinateur connecté est potentiellement vulnérable. L'entité responsable s'appelle un intrus. Les motivations varient : curiosité, gain financier, concurrence, vengeance, ou même recherche de failles pour les corriger.
// pourquoi attaque-t-on ?
💰 Argent / ransomware · 🕵️ Espionnage · 😈 Vandalisme · 🏆 Challenge technique · 🔍 Bug bounty (légal)
// interne vs externe
Interne : employé mécontent, erreur humaine
Externe : hacker distant, virus propagé via email
🌐
Cybercrime
Crimes via les technologies numériques…
+
Le cyberespace est l'ensemble des données numérisées formant un univers d'information via l'interconnexion mondiale des ordinateurs. Un crime informatique utilise un système informatique comme objet ou moyen du délit. Le cybercrime fait spécifiquement appel à Internet.
// distinctions clés
Crime informatique = délit impliquant un ordi (outil ou cible)
Cybercrime = sous-ensemble utilisant Internet pour le délit
// 02 · les 5 piliers

Objectifs de la Sécurité

5 propriétés fondamentales. Chaque système sécurisé doit les garantir. Clique sur un pilier pour voir le détail, les exemples, et l'analogie qui va tout clarifier.

01
Disponibilité
▶
Garantir l'accès aux ressources avec des temps de réponse acceptables — continuité de service.
La disponibilité assure que les services sont accessibles quand on en a besoin, avec des performances acceptables. On la garantit grâce à la redondance (duplication des ressources), les sauvegardes régulières et des plans de continuité d'activité (PCA). Une panne = violation de disponibilité.
🍕 Imagine une pizzeria ouverte 24h/24. Si elle est subitement fermée 3h au moment où t'as faim, la disponibilité n'est pas assurée. En informatique : un serveur qui tombe = frustration garantie.
🔁 Redondance 💾 Sauvegardes 📊 SLA
02
Intégrité
▶
Détecter si des données ont été modifiées — intentionnellement ou accidentellement.
L'intégrité garantit que les données sont exactement celles qui ont été produites, sans altération. On la vérifie via des fonctions de hachage (SHA-256, MD5) : si un seul bit change, le hash change complètement. Elle protège contre les modifications malveillantes ET les erreurs accidentelles.
📝 Ton contrat de travail dit "salaire : 500€/mois". Si quelqu'un change "500" en "50", tu veux le savoir ! L'intégrité te dit : "ce document a été modifié" — ou "il est intact".
SHA256("Bonjour") = 7509e5bda0c762d2bac7f90d758b5b2263fa01ccbc542ab5e3df163be08e6ca9
SHA256("Bpnjour") = a3f2... ← complètement différent !
03
Confidentialité
▶
Garantir que seules les entités autorisées comprennent les informations transmises.
La confidentialité s'assure que les données ne sont lisibles que par les personnes autorisées. On l'obtient par deux moyens complémentaires : le contrôle d'accès (qui peut lire ?) et le chiffrement (transformer les données pour qu'elles soient incompréhensibles sans la clé).
🔒 C'est comme envoyer une lettre dans une enveloppe scellée. Même si le facteur la lit (interception), il ne voit que du charabia chiffré. Seul le destinataire avec la "clé" peut déchiffrer.
🔐 Chiffrement 🚪 Contrôle d'accès 👁️ Need-to-know
04
Authentification
▶
Vérifier l'identité d'une entité avant de lui accorder l'accès à une ressource.
L'authentification consiste à vérifier qu'une entité est bien celle qu'elle prétend être. Il existe 3 facteurs d'authentification : ce qu'on sait (mot de passe, PIN), ce qu'on possède (carte, token), ce qu'on est (biométrie : empreinte, visage). Le MFA (Multi-Factor Authentication) combine plusieurs facteurs.
🎫 C'est le videur d'une boîte de nuit qui vérifie ta carte d'identité + compare ta tête à la photo. "T'es bien toi ? OK, tu rentres."
🧠 Mot de passe 📱 OTP / Token 👆 Biométrie
05
Non-Répudiation
▶
Ne pas pouvoir nier qu'un événement ou une action a bien eu lieu.
La non-répudiation garantit qu'une entité ne peut pas nier avoir effectué une action. On l'obtient via les signatures numériques, les journaux d'audit (logs) et les horodatages. Elle est cruciale dans les transactions électroniques, les contrats numériques et les échanges légaux.
📦 Le bon de livraison que tu signes quand tu reçois un colis : le livreur prouve qu'il t'a bien remis le colis. Tu ne peux pas dire "je n'ai rien reçu" si ta signature est là.
✍️ Signature numérique ⏱️ Horodatage 📋 Logs

// 06 · référence rapide

Glossaire Express

Tous les termes clés du chapitre en un coup d'œil. Clique pour ouvrir la définition complète.

Cyberespace
Univers numérique formé par l'interconnexion mondiale des ordinateurs.
Menace
Action susceptible de nuire à un système dans l'absolu.
Vulnérabilité
Faille exploitable pour obtenir un accès illicite.
Intrus
Entité responsable d'une attaque sur un système.
CVE
Common Vulnerabilities and Exposures — base de données des failles connues.
Sniffer
Outil qui capture et analyse le trafic réseau (ex : Wireshark).
Botnet
Réseau de machines infectées contrôlées à distance pour des attaques DDoS.
2FA / MFA
Double / Multi-Factor Authentication — couches de vérification d'identité.
Hachage
Fonction mathématique produisant une empreinte unique d'un fichier (SHA-256, MD5).
Cheval de Troie
Malware caché dans un programme légitime qui s'exécute à l'insu de l'utilisateur.
Phishing
Usurpation d'identité par email pour voler des informations.
Zero-Day
Vulnérabilité inconnue du fabricant, donc non corrigée = danger maximal.
